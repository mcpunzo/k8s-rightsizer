# Support

## Community support

For questions, usage help, or feature discussion, use GitHub issues and discussions in the project repository.

## Bug reports

When opening a bug report, include:

- expected behavior
- actual behavior
- steps to reproduce
- logs or stack traces
- environment details

## Security issues

Do not report security issues in public issues.
See [`SECURITY.md`](SECURITY.md) for private reporting instructions.

## Commercial and enterprise inquiries

For enterprise support, proprietary plugin/connector packages, enterprise deployment, or partnership requests, contact:

- `contacts@k8srightsizer.com`
